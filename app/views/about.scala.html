About NavProt

NavProt is a web application for predicting conformational transition pathways of proteins by using a novel multiple-robot navigation algorithm. Application basically takes two inputs, initial and final conformations of the protein. It creates a coarse grain model of both conformations by considering positions of alpha-carbon atoms only. Afterwards it builds an artificial potential function. Derivative of this function results in direction vectors that lead the structure to final conformation. You can read more about the theory in this article.

How to use it

If you want to simulate transitional pathway of a protein you need two PDB files, one for first conformation and another for final conformation. You also need to know which chain you are going to use in which PDB file. In addition, the chains you submit should have the same number of residues. Even though sequences of your chains are identical, they might not have the same number of residues. In that case, you might have to leave out some number of residues from one of the chains. Therefore, we advise you to align your structures and modify your PDB files accordingly before you submit a request to the application.

How long does a simulation take?

It depends on the number of residues and the distance between initial and final conformations, it might take from a few hours up to two or three days. If there is still another job running after you submitted your request, your job is put in the queue. If you provide an e-mail address with your upload, you will receive e-mails about the status of your request. You can also check it on your results page.

If you have a structure which has more than 400 residues, we suggest you to use standalone version to run it locally since running time might be relatively long.

What is the result of the simulation?

Each conformation at each step of the simulation is recorded in a PDB file as NMR model, you can download this PDB file. After simulation is over, conformations at each step are aligned (using FATCAT rigid algorithm) with initial and final conformations and RMSD values are recorded. You can observe these values on a plot. Here is an example results page.

If you obtain a trajectory which has approximately more than 300 snapshots, visualization of the trajectory might be slow. Results page uses JSMol to show the structures and your browser might become unresponsive while compiling JavaScript. Therefore, for such cases, it might be easier for you to download the result PDB to visualize the trajectory on your own computer by using PyMOL, VMD etc.

Is it possible to run it locally?

Yes, there is a standalone version available. You can check the downloads page here.
